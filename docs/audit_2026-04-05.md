## Quality Assessment

### Which Folder Has Better-Quality Names?

**Second Brain/Plaud wins on naming discipline** (10% malformed vs 30% in FC), but FC wins on content richness:

- FC has paired `.md` + `- Recording.txt` files, meaning each recording has both a Markdown summary and the raw transcript
- SB has mainly single `.txt` files per recording
- FC uses readable human topic titles; SB uses underscored machine-readable names with source category

Neither format is ideal as-is. The goal format should be: `YYYY-MM-DD - Topic Title.md` (summary) and `YYYY-MM-DD - Topic Title - Recording.txt` (transcript/audio), consistent with the FC email-pipeline output for well-formed files.

## Consolidation Plan

### Canonical Folder: `01 - Second Brain/Plaud`

**Rationale:** This is already integrated into the Second Brain organizational structure. The `Filing Cabinet/Plaud` folder was created by the automation as an intermediate destination and should be treated as a staging area, not a permanent home.

### Step 1: Normalize Malformed Files Before Moving

Before any consolidation, malformed files need to be addressed. Do not consolidate garbage names into the canonical folder.

#### FC Malformed (priority order):

1. **173 `YYYY-MM-DD HH:MM` files** — These are the backfill batch. They have real dates but wrong format. A script can rename them: strip the HH:MM timestamp and use the `[Plaud-AutoFlow]` portion as the title if present, or the summary type (e.g. `summary-Meeting Minutes.txt`). Group by HH:MM cluster to associate summaries with their recording.

2. **23 `MM-DD` no-year files** — These are from December 2025 (the MM values are 12, context confirms this). Prepend `2025-` to fix.

3. **20 weekday+garbled-time files** — Files like `2025-10-01 - Isha - Monday at  AM.md`. The date portion is correct; the time part in the subject was malformed during email processing. These are workable — keep the `YYYY-MM-DD - Topic` prefix and strip the ` - Weekday at TIME` suffix, leaving the topic.

4. **17 no-date files** — Manual review required. Examples: `Dec 18 at 8-21 PM`, `Team Call and Colin`. Assign dates based on content or creation time metadata. Low priority.

#### SB Malformed:

1. **26 `MM-DD` no-year files** — January 2026 context. Prepend `2026-`.

2. **23 `None - Source - Topic_(NoDate)` files** — Use Drive `createdTime` metadata to assign dates where possible; otherwise flag for manual review.

3. **6 `YYYY-MM-DD HH:MM` files** — Same fix as FC: strip timestamp, use topic if present.

### Step 2: Deduplicate the 13 Shared-Date Overlaps

For each of the 13 dates where both folders have files, do a content comparison:
- If the files cover the same recording (same topic, same duration), keep whichever has the richer content (prefer `.md` with summary over raw `.txt`).
- If the files cover different recordings on the same day (common — multiple recordings per day), keep all of them.
- The one exact-name duplicate (`2026-03-06 - Meeting- Flex Solutions Charter...`) should be deduplicated: keep the SB copy (already in canonical location), delete the FC copy.

Automated approach: compare file sizes and first 200 chars of content for same-name/same-day files.

### Step 3: Move FC-Only Files to SB

After normalization, move all FC files that don't have a match in SB to `01 - Second Brain/Plaud`. Use the Drive API `files().update(fileId, addParents, removeParents)` to move without re-uploading.

Priority order:
1. Well-formed `YYYY-MM-DD - Topic` files from 2025-12 through 2026-04 (the bulk of unique FC content)
2. Fixed MM-DD files
3. Fixed HH:MM files
4. Deferred: no-date files pending manual review

### Step 4: Retire Filing Cabinet/Plaud

Once all content is consolidated:
- Verify SB Plaud folder file count equals expected total
- Move any remaining FC files to `Filing Cabinet/Plaud/Archive` for 30-day hold
- Update `automation.py` to target `01 - Second Brain/Plaud` instead of `Filing Cabinet/Plaud`
- Update `drive_tree.json` refresh to confirm new canonical path

### Step 5: Prevent Future Divergence

- Update `automation.py`: change `get_or_create_folder("Filing Cabinet/Plaud")` to `get_or_create_folder("01 - Second Brain/Plaud")`
- Once direct API integration is live (see `api_integration_plan_2026-04-05.md`), the Gmail email pipeline will be deprecated and this will be a non-issue
